# 전략 실패 시나리오 및 한계

이 문서는 각 전략이 **어떤 시장 조건에서 깨지는지**를 명시한다.
백테스트 수익률이 좋아도 아래 레짐에서는 실거래 투입을 재검토해야 한다.

---

## 1. 베이시스 평균회귀 (Basis Mean Reversion)

### 전략 가정
- 스팟-선물 베이시스는 단기적으로 평균 주변을 오간다 (OU/평균회귀).
- 극단적 z-score 에서 반대 포지션을 취하면 회귀 수익을 얻는다.

### 깨지는 조건

| 시나리오 | 메커니즘 | 증상 |
|---------|---------|------|
| **장기 콘탱고** | 기관/ETF 수요로 선물 프리미엄이 구조적으로 유지 | z-score 가 장기간 양(+) → 숏 진입 후 손실 누적 |
| **장기 백워데이션** | 공급 과잉·청산 캐스케이드로 선물 디스카운트 지속 | z-score 가 장기간 음(-) → 롱 진입 후 손실 |
| **반감기 붕괴** | half_life 필터를 통과해도 레짐 전환 시 회귀 속도 급변 | stop_zscore 손절 반복, 수수료·슬리피지 누적 |
| **펀딩비 역방향** | 포지션 방향과 펀딩비가 장기간 불리 | 펀딩 정산으로 알파 상쇄 |
| **거래소 이벤트** | 현물-선물 가격 괴리 급확대 (유동성 위기) | 베이시스 계산 자체가 비정상, 급격한 MDD |

### 모니터링 지표
- 베이시스 90일 이동평균 대비 현재 z-score 부호가 30일 이상 동일 → 레짐 전환 의심
- half_life 가 max_half_life 에 근접하는 빈도 증가 → 전략 비활성화 검토

## 2. 페어 트레이딩 (BTC-ETH Cointegration)

### 전략 가정
- BTC-ETH 로그가격이 공적분 관계에 있다.
- 스프레드 z-score 극단에서 회귀에 베팅한다.

### 깨지는 조건

| 시나리오 | 메커니즘 | 증상 |
|---------|---------|------|
| **공적분 붕괴** | ETH/BTC 상대 강도 레짐 전환 (예: 알트시즌 → 비알트시즌) | coint p-value > 0.05, 진입 중단 또는 손절 |
| **단방향 추세** | 한쪽만 급등/급락 (ETF, 규제, 해킹) | 스프레드가 회귀하지 않고 발산 → stop_zscore 손절 |
| **유동성 비대칭** | ETH 슬리피지가 BTC 대비 커짐 | 헤지 레그 손실이 메인 레그 수익 초과 |
| **상관 급변** | 매크로 쇼크 시 모든 코인 동조화 | z-score 신호는 있으나 스프레드 미회귀 |

### 모니터링 지표
- 공적분 재검정 실패 연속 횟수
- IS-OOS Sharpe 괴리 > 1.0 (워크포워드 경고)

## 3. 펀딩비 캐리 (Funding Carry, Delta Neutral)

### 전략 가정
- 현물 롱 + 무기한선물 숏으로 델타를 0에 가깝게 유지한다.
- 펀딩비가 양(+)이고 연환산 임계값 이상일 때 진입하여 8시간마다 펀딩을 수취한다.

### 깨지는 조건

| 시나리오 | 메커니즘 | 증상 |
|---------|---------|------|
| **펀딩비 장기 마이너스** | 약세장·백워데이션에서 숏이 펀딩을 지급 | 구조적 손실, 알파 없음 |
| **펀딩비 급락** | 수요 급변·청산 캐스케이드 | 진입 후 펀딩 수익 < 거래비용 |
| **숏 레그 청산** | BTC 급등 시 마진 부족 | 강제 청산 시뮬레이션 발동, 대형 손실 |
| **거래비용 누적** | 현물+선물 4레그 수수료·슬리피지 | 낮은 펀딩 환경에서 순수익 마이너스 |
| **베이시스 확대** | 캐리 포지션과 무관하게 베이시스 발산 | 델타 헤지 불완전 시 잔여 노출 손실 |

### 모니터링 지표
- 연환산 펀딩비 30일 이동평균 < exit 임계값 → 청산 검토
- 숏 레그 미실현 손실 / 마진 비율 → 청산 위험 근접 시 포지션 축소

## 4. 공통 리스크 (리스크 매니저 한계)

- **일일 손실 한도 (3%)**: 단일 전략·단일 포지션 기준. 급변 시 4h 봉 내 한도 초과 가능 (봉 내 고저 미반영).
- **동시 포지션 제한**: 현재 단일 전략 실행 구조. 멀티 전략 병행 시 별도 집계 필요.
- **슬리피지 모델**: 고정 bps. 변동성 급등 시 실제 슬리피지가 모델보다 클 수 있음.
